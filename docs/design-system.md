# Design System

## Product character

ParseLayer should feel precise, calm, premium, and trustworthy. Operational clarity is more important than decorative AI styling.

## Foundations

Define semantic tokens for colour, typography, spacing, radius, borders, elevation, iconography, motion, and responsive breakpoints. Branding remains replaceable until the final identity is approved.

## Core components

- buttons and links
- text, select, date, and file inputs
- tables and data grids
- navigation and breadcrumbs
- tabs and filters
- drawers and dialogs
- alerts, toasts, and banners
- loading, empty, error, and retry states
- status and confidence indicators
- document viewer controls
- field editor and evidence highlight patterns

## Document review pattern

Large screens use a resizable split view with the document as a primary surface and extracted fields beside it. Selecting a field highlights its source evidence. Small screens use an intentional source/data switch rather than compressing both panes.

## Semantic states

Users must distinguish high confidence, needs attention, missing, inferred, corrected, validation warning, processing, and failure states without relying on colour alone.

## Accessibility

Target WCAG 2.2 AA. Support keyboard navigation, visible focus, adequate contrast, clear labels, error association, reduced motion, screen-reader announcements, and practical touch targets.

## Visual rules

Avoid excessive gradients, decorative charts, huge empty cards, repeated containers, random colours, tiny text, hidden critical actions, and unnecessary motion. Use motion only to communicate state or spatial relationships.
